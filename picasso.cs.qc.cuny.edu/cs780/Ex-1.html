Keitaro Yukawa, Department of Computer Science, Queens College, CUNY

Exercise Set #1

1. Let R(A, B) be any binary relation. Prove each of the following:
  1. For each b ∈ B, R↓A(b) = R¯↓A(b).
  2. For each a ∈ A, R↓B(a) = R¯↓B(a).
2. For each of the following relations, describe in clear English the meaning of the given projections.
  1. teach(Professor, CourseSection)
    teach(p, c) means professor p taught course section c.
    1. teach↓CourseSection(p)
    2. teach↓Professor(c)
  2. child(Person, Person)
    child(p1, p2) means person p1 is a child of person p2.
    1. child↓Person1(p)
    2. child↓Person2(p)
    3. child↓Person2
  3. sibling(Person, Person)
    sibling(p1, p2) means person p1 is a sibling of person p2.
    1. sibling↓Person1(p)
    2. sibling↓Person2(p)
    3. sibling↓Person1
  4. employment(Employee, Office, JobPosition)
    employment(e, o, j) means employee e works for office o in job position j.
    1. employment↓Employee(o, j)
    2. employment↓Office(e, j)
    3. employment↓JobPosition(e, o)
    4. employment↓Employee(o), where o is an arbitrary but fixed office
  5. contract(Author, Publisher, Book)
    contract(a, p, b) means author a signed a contract with publisher p to write book b.
    1. contract↓Author(p, b)
    2. contract↓Publisher(a, b)
    3. contract↓Book(a, p)
    4. contract↓⟨Author, Publisher⟩
    5. contract↓Author(p), where p is an arbitrary but fixed publisher
3. Let child and sibling be the relations as defined in the previous exercise. The meaning of the composition (child child) is described by: (child child)(p1, p2) means person p1 is a grandchild of person p2. Following this manner, describe the meaning of each of the following compositions.
  1. child child child
  2. child sibling
  3. sibling child
  4. sibling sibling
4. Consider the following five binary relations:

teach(Professor, CourseSection)
teach(p, c) means professor p taught course section c.

offerCourse(Department, CourseSection)
offerCourse(d, c) means department d offered course section c.

offerDegree(Department, Degree)
offerDegree(d, dg) means department d offered degree dg.

take(Student, CourseSection)
take(s, c) means student s completed course section c.
(Note that this binary relation can be obtained from the 4-ary relation take(Student, CourseSection, Program, Grade) in Course Notes #2 by projection: take↓⟨Student, CourseSection⟩.)

receive(Student, Degree)
receive(s, d) means student s received degree d.

a. Describe the meaning of each of the following compositions. (It will be helpful to draw a "block diagram" of these relations.)
  1. take teach¯
  2. teach take¯
  3. take offerCourse¯
  4. offerCourse take¯
  5. teach offerCourse¯
  6. offerCourse teach¯
  7. receive offerDegree¯
  8. take offerCourse¯ offerDegree
  9. teach take¯ receive
b. Define each of the following binary relations by relation composition.
  1. { ⟨d, c⟩ : department d offered a degree received by a student who took course section c }
  2. { ⟨d, dg⟩ : department d offered a course section taken by a student who received degree dg }
  3. { ⟨s, s′⟩ : student s took a course section offered by a department that offered a degree received by student s′ }
  4. { ⟨c, d⟩ : course section c was taken by a student who received degree d }
  5. { ⟨c, d⟩ : course section c was offered by a department that offered degree d }
  6. { ⟨s, s′⟩ : students s and s′ took a common course section }
  7. { ⟨p, p′⟩ : professors p and p′ taught a common course section }
  8. { ⟨c, c′⟩ : course sections c and c′ were offered by a common department }
  9. { ⟨d, d′⟩ : degrees d and d′ were received by a common student }
5. Consider the 4-ary relation take(Student, CourseSection, Program, Grade) in Course Notes #2. Describe the meaning of each of the following.
  1. take↓⟨CourseSection, Program⟩(s, g)
  2. take↓⟨CourseSection, Program⟩
  3. take↓Student(p), where p is an arbitrary but fixed program
  4. offerCourse (take↓⟨CourseSection, Program⟩)
6. A binary relation R(S, S) is symmetric if
∀x,y∈S ( R(x, y) ⇒ R(y, x) ).
  1. Prove: For each symmetric relation R(S, S) and for each x ∈ S, R↓S1(x) = R↓S2(x).
  2. Find three real-world examples of symmetric binary relations.
7. A binary relation R(S, S) is transitive if
∀x,y,z∈S ( R(x, y) ∧ R(y, z) ⇒ R(x, z) ).
Find three real-world examples of transitive binary relations.

8. Prove: For any R(S, S), R is symmetric iff R = R¯.

9. Prove: For any R(S1, S2), RR¯ is symmetric.

10. Prove: For any transitive R, R¯ is transitive.

11. Compare relational and object-oriented DBs and point out three relative advantages of each.

12. Relational DBs are said to be "value oriented" while object-oriented DBs are said to be "reference oriented". Concisely explain the meaning of these.

13. Concisely describe each of the following:
  1. data definition language
  2. data manipulation language
  3. application development language
14. Describe one major difference between relational and object-oriented DBs with respect to how the three types of languages in Q. 13 are separated or combined within the context of database applications development.

15. Name the database component responsible for each of the following:
  1. Manages low-level, physical data structures for data schemas, actual data objects, and indexes.
  2. Controls concurrent access to the database.
  3. Processes queries and translate/optimize them into efficient algorithmic forms.
  4. Manages recovery from system failures.
16. Which of the following correctly represent characteristics of (i) transparent persistent object store approach, (ii) object-relational mapping approach, and (iii) autonomous object store approach? Choose all that apply.
  1. A real object store exists.
  2. Only a virtual object store exists.
  3. The view of the object store from the ADL is always transparent.
  4. The view of the object store may or may not be transparent.
  5. The database system provides a full object-oriented database language that supports all of DDL, DML, and ADL.
  6. Database applications development needs to commit to a single programming language.
  7. Good fit for multi-language, heterogeneous environments.
  8. There is potential inefficiency incurred by join operations on back-end tables that implement relations between classes.
