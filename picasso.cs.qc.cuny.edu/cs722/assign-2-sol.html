  1. Consider the NTM in Question 2 here.

    1. Find the exact formula of the worst-case time complexity function, WM(n), of this NTM. Justify your answer.

      First, we note that M never changes the input string, so we can derive WM(n) based solely on the input string.

      Once M enters q3, M can never move the tape head to right as q3 and q4 have no transitions with the head moving to right, and in this case, M can do at most three transitions q0 → q3 → q4 → qaccept. All the other transitions move the head to right, so M can do at most n R-transitions ending at the blank symbol. But if M reads the blank symbol, there are no further transitions. Hence the worst case would occur when the head moves to right all the way to the last symbol of the input string, then entering q3 and reading "cba" backward, performing (n−1)+3 = n+2 transitions. This occurs for all and only strings of the form wabc, where w ∈ Σ*, for which WM(n) = n+2, n ≥ 3. For n = 0, 1, 2, exhaustive inspection of the strings of these lengths shows that WM(n) = n if n = 0, 1 and WM(n) = n+1 if n = 2.

  2. Consider the following verifier V for the language L = { wcabx } ∪ { wabcx }, where Σ = { a, b, c }, w, x ∈ Σ*, which is the language decided by the NTM in Question 1. A proposed certificate encodes a pair ⟨n1, n2⟩ where n1 is a non-negative integer in unary notation and n2 = 0 or 1. 
    step 1: By consulting n1, scan the initial n1 symbols on tape.
        If the tape head reaches "#" in the scan because n1 > the input-string length, reject.
step 2: Read n2.
        If n2 = 0, check if the next three tape symbols are "cab";
        if so, accept, o.w., reject.
        If n2 = 1, check if the previous three tape symbols are "cba" backward;
        if so, accept, o.w., reject.
    1. Prove that V verifies L.

      We have to prove L = L(V).

      Suppose v ∈ L. Then v = wcabx or wabcx, where w, x ∈ Σ*.
      Case 1: v = wcabx. Let n1 = |w|, n2 = 0. When V runs on ⟨v, ⟨n1, n2⟩⟩, it scans w in step 1 then detects the next three tape symbols cab in step 2, and accepts, so v ∈ L(V).
      Case 2: v = wabcx. Let n1 = |wabc|, n2 = 1. When V runs on ⟨v, ⟨n1, n2⟩⟩, it scans wabc in step 1 then detects the previous three tape symbols cba backward in step 2, and accepts, so v ∈ L(V).

      Conversely, suppose v ∈ L(V). Then ∃c s.t. V accepts ⟨v, c⟩.
      Case 1: c = ⟨n1, 0⟩. V scans the initial n1 tape symbols then detects the next three symbols cab. So v must be of the form wcabx where |w| = n1, hence v ∈ L.
      Case 2: c = ⟨n1, 1⟩. V scans the initial n1 tape symbols then detects the previous three symbols cba backward. So v must be of the form wabcx where |wabc| = n1, hence v ∈ L.

    2. By the method used in the proof of Theorem VtoN, define an NTM equivalent to V.

      Let f(n) be a function as described in Theorem VtoN. The equivalent NTM, M, is defined as follows.

      Given input string w:

  4. Prove each of the following.

    These can be proved by NTMs or verifiers due to their essential equivalence. The following uses verifiers.

    1. If A, B ∈ NP, then A ∪ B ∈ NP.

      Let VA, VB be polynomial-time verifiers for A and B with WVA(n) = O(nk1) and WVB(n) = O(nk2). We will define a polynomial-time verifier VA ∪ B for A ∪ B using VA and VB.

      Definition of VA ∪ B

      Given ⟨w, c⟩:
      1. Create a 2nd copy of ⟨w, c⟩ and append it to the original ⟨w, c⟩. The original ⟨w, c⟩ and the 2nd copy can be separated by a special tape symbol like "$".
      2. Run VA on the 2nd copy of ⟨w, c⟩; the original ⟨w, c⟩ is kept intact.
      3. Erase all work VA did on tape by writing blank symbols and move the tape head back to the leftmost position.
      4. Run VB on the original ⟨w, c⟩.
      5. if VA or VB accepts ⟨w, c⟩ then accept else reject

      Since VA ∪ B runs VA and VB on ⟨w, c⟩, we assume without loss of generality that |c| ≤ max(WVA(n), WVB(n)).

      We now prove VA ∪ B verifies A ∪ B, that is,
      A ∪ B = L(VA ∪ B) = { w | ∃c s.t. VA ∪ B accepts ⟨w, c⟩ }.
      Suppose w ∈ A ∪ B. Then w ∈ A or w ∈ B, so ∃cA s.t. VA accepts ⟨w, cA⟩ or ∃cB s.t. VB accepts ⟨w, cB⟩. Note that cA and cB may be distinct. In the former case, VA ∪ B accepts ⟨w, cA⟩, hence w ∈ L(VA ∪ B). In the latter case, VA ∪ B accepts ⟨w, cB⟩, hence w ∈ L(VA ∪ B). In either case, w ∈ L(VA ∪ B).

      Conversely, suppose w ∈ L(VA ∪ B). Then ∃c s.t. VA ∪ B accepts ⟨w, c⟩. This means, by definition of VA ∪ B, VA or VB accepts ⟨w, c⟩. Hence w ∈ A or w ∈ B and so w ∈ A ∪ B.

      Now the runtime analysis of VA ∪ B.

      • Step 1 can be done in O( (|w|+|c|)2 ) = O( (n+max(WVA(n), WVB(n)))2 ).
      • Step 2 takes WVA(n).
      • Step 3 can be done in O(WVA(n))+O(|w|+|c|) as no DTM can write more than the worst-case runtime number of symbols on tape.
      • Step 4 takes WVB(n).

      Since WVA(n) = O(nk1) and WVB(n) = O(nk2), the total runtime of Step 1 through Step 4 is polynomially bounded.

      In this question we provided low-level, nuts-and-bolts operations like copying ⟨w, c⟩ and erasing tape symbols. Although this kind of operation is also necessary for (b) and (c), we will omit it as the essential idea of definitions and runtime analysis is not affected.

    2. If A, B ∈ NP, then A ∩ B ∈ NP.

      Let VA, VB be polynomial-time verifiers for A and B with WVA(n) = O(nk1) and WVB(n) = O(nk2). We will define a polynomial-time verifier VA ∩ B for A ∩ B using VA and VB.

      VA ∩ B uses a 2-component certificate c = ⟨cA, cB⟩.

      Definition of VA ∩ B

      Given ⟨w, c = ⟨cA, cB⟩⟩:
      1. Run VA on ⟨w, cA⟩.
      2. Run VB on ⟨w, cB⟩.
      3. if VA accepts ⟨w, cA⟩ and VB accepts ⟨w, cB⟩ then accept else reject

      Since VA ∩ B runs VA on ⟨w, cA⟩ and VB on ⟨w, cB⟩, we assume without loss of generality that |cA| ≤ WVA(n) and |cB| ≤ WVB(n).

      We need to show:
      A ∩ B = L(VA ∩ B) = { w | ∃c s.t. VA ∩ B accepts ⟨w, c⟩ }.
      Suppose w ∈ A ∩ B. Then w ∈ A and w ∈ B, so ∃cA s.t. VA accepts ⟨w, cA⟩ and ∃cB s.t. VB accepts ⟨w, cB⟩. Let c = ⟨cA, cB⟩. Then VA ∩ B will accept ⟨w, c = ⟨cA, cB⟩⟩.

      Conversely, suppose VA ∩ B accepts ⟨w, c = ⟨cA, cB⟩⟩. By definition of VA ∩ B, this means that VA accepts ⟨w, cA⟩ and VB accepts ⟨w, cB⟩. Hence, w ∈ A and w ∈ B, and thus w ∈ A ∩ B.

      Since |cA| ≤ WVA(n) = O(nk1) and |cB| ≤ WVB(n) = O(nk2), |c| = |⟨cA, cB⟩| = O(nk1) + O(nk2)), which is polynomially bounded. Thus, consulting/extracting the two components of c can be done in polynomial time. Step 1 takes WVA(n) = O(nk1) and Step 2 takes WVB(n) = O(nk2). Thus, the total runtime is polynomially bounded.

    3. If A, B ∈ NP, then A · B ∈ NP, where A · B = { xy | x ∈ A and y ∈ B }.

      Let VA, VB be polynomial-time verifiers for A and B with WVA(n) = O(nk1) and WVB(n) = O(nk2). We will define a polynomial-time verifier VA·B for A · B using VA and VB.

      VA·B uses a 3-component certificate c = ⟨csp, cA, cB⟩. The string csp encodes a split position of the input w. If csp = i, 0 ≤ i ≤ n, w = w1 ··· wn will be split into x = w1 ··· wi and y = wi+1 ··· wn.

      Definition of VA·B

      Given ⟨w, c = ⟨csp, cA, cB⟩⟩:
      1. Split w into xy by consulting the split point encoded in csp.
      2. Run VA on ⟨x, cA⟩.
      3. Run VB on ⟨y, cB⟩.
      4. if VA accepts ⟨x, cA⟩ and VB accepts ⟨y, cB⟩ then accept else reject

      Since VA·B runs VA on ⟨x, cA⟩, runs VB on ⟨y, cB⟩, and |x| ≤ n, |y| ≤ n, we assume without loss of generality that |cA| = O(nk1) and |cB| = O(nk2).

      We need to show:
      A · B = L(VA·B) = { w | ∃c s.t. VA·B accepts ⟨w, c⟩ }.
      Suppose w ∈ A · B. By definition of A · B, w = xy, x ∈ A, and y ∈ B. So ∃cA s.t. VA accepts ⟨x, cA⟩ and ∃cB s.t. VB accepts ⟨y, cB⟩. Let csp encode the split position of w = xy, and let c = ⟨csp, cA, cB⟩⟩. Then VA·B will accept ⟨w, c = ⟨csp, cA, cB⟩⟩.

      Conversely, suppose VA·B accepts ⟨w, c = ⟨csp, cA, cB⟩⟩. By definition of VA·B, this means that w is split into xy using the split position encoded in csp, VA accepts ⟨x, cA⟩, and VB accepts ⟨y, cB⟩. Hence, x ∈ A and y ∈ B, and thus w = xy ∈ A · B.

      A split point ranges from 0 to n and can be encoded by a string of length at most n, so |csp| ≤ n. Also, |cA| = O(nk1) and |cB| = O(nk2). Hence |c| = O(n + O(nk1) + O(nk2)), which is polynomially bounded. Thus, consulting/extracting the three components of c can be done in polynomial time. Step 1 can be done in polynomial time of n = |w|. Since |x| ≤ n and |y| ≤ n, Step 2 takes at most O(nk1) and Step 3 takes at most O(nk2). Thus, the total runtime is polynomially bounded.
